هذا تحليل متكامل لمتطلبات موقع شركة **Shadj (شدج)** للجرافيك، تم إعداده بشكل منظم ليكون بمثابة دليل فني (Technical Analysis) واضح للمطورين والمصممين لتنفيذ المشروع بدقة.
## 1. الهوية البصرية والـ Splash Screen
 * **الألوان الأساسية:** يعتمد الموقع على 4 ألوان رئيسية (تم تحديد لونين منها بشكل صريح):
   * **البيج (Beige):** يعطي طابعاً فخماً واحترافياً.
   * **الأبيض (White):** للخلفيات والمساحات المريحة للعين.
   * *ملاحظة: يجب اختيار اللونين المتبقيين (مثل الأسود الفخم أو الكحلي الداكن للنصوص والعناصر الحيوية) لضمان تباين عالي (Contrast).*
 * **شاشة الترحيب (Splash Screen):**
   * **الفكرة:** رسامون يقومون برسم كلمة "شدج".
   * **الأداة المستهدفة:** تصميم الحركة عبر **Canva** (أو تحويلها إلى صيغة Lottie/GIF خفيفة لضمان سرعة التحميل).
   * **المدة:** 5 ثوانٍ.
   * **الميزة الأساسية:** إضافة زر **تخطي (Skip)** يظهر فوراً لمنع شعور المستخدم بالملل في الزيارات المتكررة.
## 2. الهندسة التقنية للموقع (Technical Stack)
 * **الخلفية البرمجية (Backend):** بيئة عمل **PHP** (يُفضل استخدام **Laravel** كإطار عمل لضمان الحماية الشديدة وإدارة لوحة التحكم بسلاسة).
 * **الواجهة الأمامية (Frontend):** متطلبات جرافيكس عالي وأنيميشن متطور.
   * *التقنيات المقترحة:* HTML5, CSS3 (مع مكتبات أنيميشن مثل GSAP أو Framer Motion)، و JavaScript (أو Vue.js/React لربط الحركات بسلاسة).
 * **نظام تسجيل الدخول (Authentication):**
   * ربط الـ API الخاص بـ **Google Login**.
   * ربط الـ API الخاص بـ **Apple Login** (يتطلب حساب مطور من أبل وشروط حماية خاصة).
## 3. هيكلة صفحات الواجهة الأمامية (Frontend Pages)
### أ. الصفحة الرئيسية (Home Page)
 * **القسم الترحيبي (Hero Section):** يعكس قوة الجرافيكس والأنيميشن بالشركة.
 * **قسم "أعمالنا" (Portfolio):**
   * عرض كافة البوسترات التي صممتها الشركة.
   * **ميزة التصنيف:** ذكر اسم الشركة وصاحب كل بوستر وفقاً لنوع التصميم (مثل: تصاميم سوشيال ميديا، هويات بصرية، لوحات إعلانية).
### ب. صفحة من نحن (About Us)
 * قصة شركة "شدج"، رؤيتها، والفريق القائم عليها بأسلوب بصري متميز.
### ج. صفحة ابدأ التصميم (Start Designing / Order Form)
 * صفحة ديناميكية تتيح للعميل طلب تصميم (إدخال البيانات، رفع الملفات والمراجع، اختيار نوع البوستر، وتحديد المتطلبات).
### د. صفحة تسجيل الدخول (Login Page)
 * واجهة بسيطة تحتوي على خيارات التسجيل السريع (Oauth) عبر Google و Apple.
## 4. لوحة تحكم الأدمن (Admin Dashboard)
تم تصميم هذا الجزء ليعطيك تحكماً كاملاً ومراقبة دقيقة لكل ما يحدث داخل الموقع:
### أ. إدارة الصلاحيات (User Management & Roles)
 * التحكم في رتب الحسابات (أدمن، مصمم، كاتب).
 * تحديد **من يملك صلاحية إضافة البوسترات** وتعديلها على الصفحة الرئيسية.
### ب. نظام المراقبة الفورية والتحليلات (Analytics & User Tracking)
 * **العداد اللحظي:** عرض عدد الزيارات الحالية للموقع (Active Users).
 * **سجل تحركات المستخدم (User Journey Logs):** تتبع تفصيلي لكل زائر منذ لحظة دخوله وحتى خروجه (مثال: *دخل الرئيسية -> انتقل لصفحة أعمالنا -> ضغط على بوستر شركة X -> خرج من الموقع*). يتم تنفيذ ذلك عبر نظام "Logs" مخصص في الـ Backend أو ربط مخصص مع أدوات مثل Mixpanel أو Hotjar.
## جدول تلخيصي للمشروع (Project Matrix)
| الجزء | التقنية / الأدوات | الوظيفة الرئيسية |
|---|---|---|
| **الـ Splash Screen** | Canva + JS Skip Button | عرض أنيميشن الرسم لمدة 5 ثوانٍ مع خيار التخطي. |
| **الفرونت اند (Frontend)** | HTML/CSS/JS (Advanced Graphics) | عرض الهوية باللون البيج والأبيض، وقسم أعمالنا المصنف. |
| **الباك اند (Backend)** | PHP (Laravel) | إدارة البيانات، الأمان، والربط مع لوحة التحكم. |
| **التسجيل** | OAuth (Google & Apple APIs) | تسجيل دخول آمن وسريع بنقرة واحدة. |
| **لوحة التحكم** | Custom PHP Dashboard + Tracking system | مراقبة الزوار خطوة بخطوة، عداد لحظي، وإدارة صلاحيات النشر. |
### نصيحة فنية لتنفيذ الجرافيكس العالي:
بما أن الموقع يطلب "جرافيكس عالي"، يجب التنبيه على المطورين بعمل **Optimization (تحسين وضغط)** لجميع الصور والبوسترات وصيغ الأنيميشن (باستخدام صيغ مثل WebP للصور و WebM للفيديوهات)، وذلك لكي لا تؤثر الجرافيكس العالية على سرعة تحميل الموقع وتجربة المستخدم.
